## Chap.0\_ IT 기획이란?

### 1. IT 프로젝트

소프트웨어 개발은 사용자의 니즈를 해결하기 위해 소프트웨어를 만들어 문제를 해결하는 방법이다. (사용자라 하면 일반인, 기업 등 소프트웨어를 사용하는 대상자를 의미한다.)

그럼 흔히 말하는 "IT 기획이란" 무엇일까?

#### 1.1 소프트웨어 개발 프로세스

소프트웨어 개발 프로세스란 일반적으로 다음과 같이 순서대로 이루어지는 것을 의마한다.
(소프트웨어 개발의 성패에 중요한 영향을 미친다!)

1. 요구사항 분석 (Requirement Analysis)
   여기 단계에서는 프로젝트 목적 수립, 요구사항 정의 및 분석을 총 틀어서 한다.
   이 프로젝트가 왜 필요하고, 프로젝트에는 어떤 적이 들어가야 하는지를 정하는 단계!

2. 기획 (Planning)
   기획 단계에는 요구사항 분석 단계의 산출물을 기반으로 프로젝트의 범위, 예산, 일정 등을 정한다.
3. 설계 (Design)
   소프트웨어의 구조와 동작 방식을 설계한다.
4. 개발 (Implementation)
   설계된 내용을 기반으로 개발을 한다.
5. 테스트 (Testing)
   개발된 소프트웨어가 예상대로 동작하는지 확인하는 단계이다.
6. 배포 (Deployment)
   사용자가 실제 환경에서 소프트웨어를 사용할 수 있도록 한다.
7. 유지보수 (Maintenance)
   배포 후 소프트웨어를 지속적으로 개선한다. 기존에는 못 발견한 오류를 해결하거나, 새로운 요구사항을 반영해야 할 수 있다.

~~정의 -> 분석/설계 -> 개발 -> 테스트 -> 배포 -> 유지보수~~
즉, IT기획은 여기서 두 번째 단계를 의미하는 것이다.

### 2. 다양한 IT 직군

IT 프로젝트가 진행되면 이런 직군(직무)의 사람들이 같이 일하게 된다.

- 기획 직군
  (전략 기획, 사업 기획, 서비스 기획 등)
- 디자인 직군
  (UI 디자이너, UX 디자이너, 그래픽 디자이너 등)
- 개발 직군
  (프론트엔드, 앱 개발자, 백엔드, 웹 퍼블리셔, 데브옵스 엔지니어, QA 엔지니어 등)

- IT 제품 관리 직군
  (PO, PM, PL 등)
- 기타
  (데이터 분석가, 보안 엔지니어, 머신러닝 엔지니어 등)

PM(Product Manager, Project Manager)는 제품 제작이나 프로젝트가 잘 완성되도록 전반적으로 관리하는 사람이다.
PM은 보통 각 직군의 리더인 PL과 함께 일하고, 각 직군은 SDLC의 단계에 맞게 일을 한다.

기획쪽 직무가 아니더라도 점점 임원이 되고 경영진이 되면 기획을 수행하는 책임이 느는 것 같다.
